🧠 Why This Exists

Most dashboards fail quietly.

Not because they look bad —
but because the data behind them cannot be trusted.

This project focuses on the part most people ignore:

👉 building a system where every number can be trusted

⸻

⚡ What This Actually Is

This is not just a Power BI dashboard.

It is a full data system:

Raw Data → Cleaning → Structuring → Validation → Insight Layer

Where:
	•	data is cleaned multiple times
	•	assumptions are tested
	•	and outputs are verified before visualisation

🔬 Data Engineering Layer (Where the real work happens)

This is where most dashboards break.

This system handles:
	•	Inconsistent IDs
	•	Missing values
	•	Duplicate records
	•	Category mismatches
	•	Silent data errors

👉 Through iterative cleaning cycles

Clean → Build → Validate → Break → Fix → Repeat
